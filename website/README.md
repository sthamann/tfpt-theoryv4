# TFPT — Topological Fixed-Point Theory · Website

Modern, accessible, SEO-optimized presentation of the **TFPT 4.5 paper series** — boundary polarization, carrier rigidity, and observable closure — by Stefan Hamann & Alessandro Rizzo.

Built with **Next.js 16**, **Tailwind CSS v4**, **TypeScript**, **Motion**, and **KaTeX** for math rendering. Designed for one-click **Vercel** deployment.

## Stack

- **Next.js 16.2** with App Router and Turbopack
- **React 19**
- **Tailwind CSS 4.2** (CSS-first configuration)
- **TypeScript 5.9** (strict)
- **Motion 12** (Framer Motion successor) for animations
- **KaTeX 0.16** for LaTeX rendering
- **Lucide React** for icons

## Local development

```bash

```

Open <http://localhost:3000>.

```bash

```

## Deploy to Vercel

### Option A — Vercel CLI

```bash
npm i -g vercel
cd website
vercel              # follow the prompts (default settings work)
vercel --prod       # promote to production
```

### Option B — Git import

1. Push the `website/` folder to a GitHub/GitLab/Bitbucket repository.
2. Go to <https://vercel.com/new> and import the repository.
3. **Root Directory**: `website` (if it's nested in a larger repo).
4. **Framework Preset**: Next.js (auto-detected).
5. **Build & Output settings**: leave defaults (`npm run build`, output `.next`).
6. **Environment Variables** (optional): set `NEXT_PUBLIC_SITE_URL` to your final domain so the sitemap and JSON-LD use the canonical URL.
7. Click **Deploy**.

The site is fully static after build (`○ (Static) prerendered as static content`), so it serves entirely from Vercel's edge network with zero cold-start latency.

## Content & data

All paper and prediction content is read from `lib/papers.ts` and `lib/predictions.ts`. To update text, formulas, target values, or add a new prediction, edit those files — no other change is required.

PDFs in `public/papers/` and `public/predictions/` are mirrors of the corresponding `tfpt-45/` directory. Replace them in place to update the downloadable artifacts.

## Accessibility & SEO

- Skip-to-content link, focus rings, prefers-reduced-motion respected.
- Semantic landmarks: `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`.
- All interactive elements are keyboard-reachable; mobile menu is `aria-expanded`.
- OpenGraph + Twitter Card metadata + structured `ResearchProject` JSON-LD.
- `robots.txt`, `sitemap.xml` autogenerated.
- Cache-control headers for PDF downloads (`vercel.json`).
- Color contrast meets WCAG AA on the dark theme.
